High-Performance & High-Flexibility Distributed System Introduction: 2

Posted on 2015-07-25   |   In architecture design , backend services

Foreword

This is a guide book for fresh man in high-performance & high-flexibility distributed system field.

This is not a teaching material, beacuse it cannot teach a beginner to implement a high-performance service or a high-performance application server step by step.
This is just a guide. On the way a fresh man become a proficient distributed architect or system architect, it can tell the man, what will he encounter, which is the key point, and which solutions can reference to.

According to the original idea, this introduction will involve the following aspects:

  • What is the high-performance? And what is the high-flexibility?
  • Technology Evolvement: from nothing to network library, from network library to RPC framework, from RPC framework to services ecological environment.
  • The components of services ecological environment.
  • Network Engine Design
  • Network Library Design
  • RPC Framework Design
  • Services Ecological Environment Design
  • New Agile Way: super fast development & rules, base on services ecological environment.
  • The performance & capability of current projects
  • The kinds of cluster
  • Cluster Management Design
  • Load Balance
  • Decentralization
  • Stateless
  • Message Queue
  • Front-end Extension
  • Encapsulate for data layer
  • Data Router
  • Distributed Stroage
  • Distributed Key-value Strogae
  • Distributed Caches: special cache vs. general cache vs. common cache
  • Message Coordination
  • Intercontinental Coordination
  • Distributed Logger
  • Real-time Computation
  • Others
  • Last: the misconception about algorithm & Database Performance Tuning & etc.

High-Performance & High-Flexibility Distributed System Introduction: 1

Posted on 2015-07-25   |   In architecture design , backend services

Reason for the Introduction

At the beginning, I had no intention of writting an introduction to arrange and record all my encountered, all my learnt, and all my done in the field of high performance & high flexibility distributed ststems. At off-duty time, don’t talk about work.

Until the day, I saw many qustions and answers at Zhihu.com (a Chinese Quora), such as following:

Q: How to implement a high concurrency application server by using epoll?
A: The codes with epoll in a matured high-performance application server is less then one in a million.
... ...
Don't think epoll is the "key technology" of a "high-performance application server", it is only a API.
... ...
epoll is just rubbish.
... ...
Don't mention the IOCP, the Proactor pattern will rape you logic structures.
...
It is not graceful as the Reactor pattern.

The answer is probably correct, and the fact is maybe true.

BUT, it is very obviously, according to the question, the questioner is just a fresh man, he haven’t any idea in the high-performance services fields.

As the highest voted answer, I think that is very bad for a beginner. First, the answer is very inane, dogmatic and abstract. What is Reactor pattern? And what is Proactor pattern? Does a beginner know these?
Second, and this is the very important, the answer is full of language violence.
It seems like a student asks his teacher: “Could you teach me how to …”
After beaten up the student with a baseball bat, then the teacher said: “Do you still want to learn this?”
Student: “Yes, I want.”
After beat again, the teacher said: “Still want?”
Student: “…”

This don’t conform to my idea!
From my standpoint, we ought to guide the fresh man step by step. We ought to inspire them, generate their interest. Rather than frightening or discouraging them.

I had searched books and informations about how to implemtnt a high-performance distributed system, but all searched are not good enough in Chinese. So, I decide to write one.

This is the reason why I write this introduction.
